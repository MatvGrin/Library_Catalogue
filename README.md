# Library_Catalogue

Functional Requirements<br>
1.The web page shall provide a registration form for a business network.<br>
2.The form shall allow the user to enter personal information, including full name and title.<br>
3.The form shall allow the user to enter company or business name, address information, including street address, city, state/province, and postal/zip code, to enter contact information such as e-mail address and phone number.<br>
4.The form shall include a submit button to send the entered data.<br>
5.JavaScript shall automatically switch header images at a fixed time interval without user interaction.<br><br>


Non-Functional Requirements<br>
1.The web page shall be implemented using structured HTML.<br>
2.CSS styles shall be stored in an external file and connected to HTML via class selectors.<br>
3.JavaScript logic shall be stored in an external file and linked to the HTML document.<br>
4.The layout shall be centered on the page with a fixed maximum width.<br>
5.The design shall be clean and consistent.<br>
6.The code shall be readable, maintainable, and consistently formatted.<br><br>


Page Design:<br>
The page uses a centered card-style layout with a maximum width.<br>
A header section is displayed at the top as a visual banner. <br>
A clear and descriptive title is placed below the header.<br>
The form body is divided into logical sections:<br>
   - Personal information<br>
   - Company information<br>
   - Address information<br>
   - Contact information<br>
Input fields are arranged in rows, with a two-column layout used where appropriate.<br>
Each input field has a label or placeholder for clarity.<br>
Helper text is displayed for address-related fields.<br>
The submit button is located at the bottom of the form and centered.<br>
