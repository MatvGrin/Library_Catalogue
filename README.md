# Library_Catalogue

Functional Requirements
1.The web page shall provide a registration form for a business network.
2.The form shall allow the user to enter personal information, including full name and title.
3.The form shall allow the user to enter company or business name, address information, including street address, city, state/province, and postal/zip code, to enter contact information such as e-mail address and phone number.
4.The form shall include a submit button to send the entered data.
5.JavaScript shall automatically switch header images at a fixed time interval without user interaction.


Non-Functional Requirements
1.The web page shall be implemented using structured HTML.
2.CSS styles shall be stored in an external file and connected to HTML via class selectors.
3.JavaScript logic shall be stored in an external file and linked to the HTML document.
4.The layout shall be centered on the page with a fixed maximum width.
5.The design shall be clean and consistent.
6.The code shall be readable, maintainable, and consistently formatted.


Page Design:
The page uses a centered card-style layout with a maximum width.
A header section is displayed at the top as a visual banner. 
A clear and descriptive title is placed below the header.
The form body is divided into logical sections:
   - Personal information
   - Company information
   - Address information
   - Contact information
Input fields are arranged in rows, with a two-column layout used where appropriate.
Each input field has a label or placeholder for clarity.
Helper text is displayed for address-related fields.
The submit button is located at the bottom of the form and centered.
